энкодер трансформера

Интерактивный разбор пути токена от слова до контекстуализированного вектора — шаг за шагом.

шаг 1 / 6

Токенизация

Слова разбиваются на токены и превращаются в целые числа — индексы в словаре.

"The cat sat"
tokenizer ↓
"The"
id = 1996
"cat"
id = 4937
"sat"
id = 3290
Важно: реальный словарь содержит ~30 000 подслов (BPE). "playing" → "play" + "##ing". Модель работает только с числами — слов она не видит.
шаг 2 / 6

Embedding lookup

По индексу берём строку из обучаемой матрицы W_E — это статичный вектор слова.

Контекста нет. "cat" в "black cat" и "cat scan" — один и тот же вектор. Это исправит энкодер. W_E — результат обучения: слова с похожим смыслом оказались рядом в пространстве. Отсюда: king − man + woman ≈ queen.

Аналогия — двоичный счётчик: бит 0 меняется каждый шаг (быстро), бит N — каждые 2ᴺ шагов (медленно). PE делает то же самое непрерывно: dim₀ отличает соседей, dim_max — позиции на расстоянии тысяч токенов. Большое k → ещё более плавный «счётчик».
шаг 4 / 6

Проекции Q K V

Каждый токен умножается на три обучаемые матрицы и получает три роли.

x₁ "cat" d=768
↙ ↓ ↘
× W_Q × W_K × W_V
Q
query — "что я ищу?"
d_k = 64
K
key — "что предлагаю?"
d_k = 64
V
value — "что отдаю?"
d_v = 64
Параллелизм: это делается для всех токенов одновременно через матричное умножения. В BERT-base: 12 голов × 64 = 768. Каждая голова учит своё "понимание" связей.
шаг 5 / 6

Attention scores

Q_cat умножается на K всех токенов → масштабируется → softmax → веса.

score = Q · Kᵀ / √d_k    weights = softmax(scores)
cat
Результат: взвешенная сумма V-векторов всех токенов. Толщина внимания = вес. Это происходит параллельно для всех токенов и всех 12 голов одновременно.

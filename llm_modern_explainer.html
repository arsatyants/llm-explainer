шаг 2 / 7

Каузальное маскирование

Декодер запрещает каждому токену "видеть" будущее через маску в матрице внимания.

Режим: Декодер (каузальный)
разрешено заблокировано
"sat" (поз. 2) видит только "The", "cat", "sat" — не "on" и не "mat"
Реализация: заблокированным позициям добавляется −∞ к логитам перед softmax → вес = 0. Весь декодер обучается параллельно: будущее скрыто маской, а не последовательностью шагов.
шаг 3 / 7

Авторегрессия и сэмплинг

Модель генерирует по одному токену. Параметры сэмплинга меняют форму распределения.

The cat sat on the ___
1.0
→ сбалансированное распределение
все
1.00
Параметры: temperature < 1 → острее (детерминированнее); > 1 → случайнее. top-k оставляет k лучших. top-p (nucleus) оставляет минимальный набор с суммарной вероятностью ≥ p. Оба применяются вместе.

KV-кэш

При инференсе K и V для прошлых токенов не меняются — их достаточно закэшировать один раз.

Генерируем токены: "The" → "cat" → "sat" → "on" → "mat"

кэш — не вычислять вычислить сейчас ещё не нужен
Выигрыш: без кэша генерация N токенов → O(N²) вычислений. С кэшем → O(N). Для контекстов 128k+ — разница в скорости в десятки раз. Цена: память GPU для хранения всех K,V.

шаг 6 / 7

Предобучение и масштабирование

LLM обучаются предсказывать следующий токен на триллионах токенов текста.

Цель обучения: cross-entropy по следующему токену
L = −(1/T) · Σ log P(xt | x1, …, xt−1)
Обучается на любом тексте без разметки — self-supervised learning.
Масштаб современных моделей (логарифмическая шкала параметров)
Законы масштабирования — Chinchilla (2022)
Оптимум: ~20 токенов на параметр
GPT-3 (175B): нужно ~3.5T токенов → обучали только 300B (недообучение)
Llama 3 8B: 8B × 20 = 160B "оптимум", но обучали 15T — намеренно переобучают для эффективного инференса
Вывод: малые модели с огромным датасетом часто бьют большие, плохо обученные.
Стоимость: предобучение Llama 3 70B ≈ 6.4M GPU-часов на H100. При $2/GPU-час ≈ $12M только на compute. Поэтому выбор архитектуры и датасета критичен.
шаг 7 / 7

Выравнивание: RLHF и DPO

Предобученный LLM умеет продолжать текст, но не "помогать". Выравнивание меняет поведение модели.

RLHF-пайплайн — нажмите на фазу для подробностей
фаза 1 / 4
Альтернатива: DPO (Direct Preference Optimization)
Убирает отдельную reward-модель и RL. Оптимизирует модель напрямую по парам предпочтений (хороший / плохой ответ).
Проще, стабильнее, часто сравнимо с RLHF. Используется в Llama 2/3, Zephyr, Mistral Instruct.
Полная цепочка современного LLM
веб-текст → токенизация → предобучение (CLM, T токенов) → SFT (инструкции) → RM + PPO/DPO → ассистент
Архитектура: Decoder-only + RoPE + RMSNorm + SwiGLU + GQA

шаг 1 / 6

От языковой модели к агенту

Голая LLM — это функция: текст→текст, один проход. Агент — это LLM в цикле с доступом к внешнему миру.

базовая LLM
text → text
Получает промпт → делает один forward pass → возвращает токены.

✓ Ответы, перефразирование, перевод
✗ Не знает текущее время
✗ Не может открыть файл
✗ Не помнит прошлые разговоры
✗ Не может проверить свой ответ
LLM-агент
цикл: think → act → observe
LLM + инструменты + цикл обратной связи.

✓ Запускает код, видит результат
✓ Читает файлы, браузер, API
✓ Долгосрочная память (вне контекста)
✓ Проверяет и исправляет себя
✓ Делегирует подзадачи другим агентам
Ключевая идея: LLM генерирует не только ответ, но и вызовы инструментов.
Среда исполняет инструмент и возвращает результат обратно в контекст.
LLM снова читает контекст → решает продолжить или ответить.

шаг 2 / 6

Tool Use / Function Calling

Модели видят список доступных инструментов в системном промпте и генерируют структурированные вызовы. Выбери инструмент:

Типичные инструменты современных агентов
⚡ bash / python — выполнение кода
🌐 web_search — поиск в интернете
📄 read_file / write_file — работа с FS
🖥 computer_use — управление ПК
🔌 MCP-серверы — любое внешнее API
🧠 memory_store — сохранить факт
Безопасность: пользователь явно разрешает каждый инструмент. Claude никогда не запустит bash без разрешения. Tool use — сознательный выбор, а не автоматическое действие.
шаг 3 / 6

Агентный цикл — ReAct

Задача: "Сколько файлов .py в этом репозитории и какой самый большой?" Нажимай для прохода по шагам.

шаг 1 / 5
Контекст модели растёт с каждым шагом:
[system prompt + tools] + [user msg] + [think₁ + act₁ + obs₁] + [think₂ + act₂ + obs₂] + … + [answer]
ReAct (Reason + Act): паттерн, при котором модель явно рассуждает перед каждым действием. Это делает цепочку прозрачной и позволяет перехватить ошибку до исполнения.
шаг 4 / 6

Расширенное мышление (Extended Thinking)

Claude 3.5+ и o1/o3 выделяют отдельный «черновик» рассуждений перед финальным ответом. Это не просто CoT в промпте — это отдельный тип токенов.

обычный режим
прямой ответ
Модель генерирует ответ за один проход. Рассуждения смешаны с ответом или отсутствуют. Быстро, но хуже на сложных задачах.
extended thinking
<thinking> + ответ
Модель генерирует скрытые «думающие» токены. Может менять направление, исправлять себя. Пользователь видит только финальный ответ (или collapse-блок).
Пример: "Докажи, что √2 иррационально"
◉ thinking ▼ раскрыть
Нужно доказать от противного.
Предположим, что √2 = p/q, где p,q — целые, gcd(p,q)=1.
Тогда 2 = p²/q² → p² = 2q².
Значит p² чётное → p чётное → p = 2k.
Подставляем: (2k)² = 2q² → 4k² = 2q² → q² = 2k².
Значит q² чётное → q чётное.
Но тогда gcd(p,q) ≥ 2 — противоречие с gcd(p,q)=1!
Следовательно, предположение неверно. √2 иррационально. ✓
Финальный ответ: Предположим от противного: √2 = p/q (несократимая дробь). Тогда p² = 2q², откуда p чётное, p=2k → q² = 2k² → q чётное. Но тогда НОД(p,q) ≥ 2 — противоречие. ∎
Thinking budget — бюджет мышления
8 000
~6 страниц рассуждений · подходит для большинства задач
Важно: thinking-токены не кэшируются между запросами и стоят дороже обычных. Claude 3.7 Sonnet: до 128k thinking tokens. Используется для математики, кода, planning.
шаг 5 / 6

Типы памяти агента

Контекстное окно — главная «рабочая память», но оно конечно. Агенты комбинируют несколько типов хранения.

50 000
In-context (рабочая)
Всё в текущем окне. Мгновенный доступ. Теряется при завершении сессии. Claude: до 200k токенов ≈ 150k слов.
External (долгосрочная)
Векторная БД, файлы, SQL. Неограниченный объём, но требует retrieval. Агент сам решает что сохранить.
Episodic (эпизодическая)
Лог прошлых разговоров и действий. Подгружается по релевантности через RAG. «Что я делал с этим файлом в прошлый раз?»
Semantic (семантическая)
База знаний: факты, предпочтения пользователя, контекст проекта. Claude Code: CLAUDE.md + memory-файлы.
Prompt caching: Anthropic кэширует часть контекста между запросами (до 5 мин). Системный промпт + инструменты + документы = до 90% экономии стоимости на повторных вызовах.

Мульти-агентные системы

Сложные задачи разбиваются между специализированными агентами. Оркестратор планирует, субагенты исполняют параллельно.

Реальные мульти-агентные паттерны
Claude Code — оркестратор + субагенты для параллельного поиска по коду
Retrieval pipeline — query → embedder → vector DB → reranker → LLM
Critic pattern — агент-генератор + агент-критик проверяет результат
Speculative execution — несколько агентов пробуют разные подходы параллельно
Полная карта современного LLM-агента
Пользователь → системный промпт + инструменты → LLM (think)
→ [tool_use JSON] → среда исполняет → [tool_result] → LLM снова
→ ... (цикл) ... → финальный ответ пользователю
+ долгосрочная память + extended thinking + мульти-агенты
MCP (Model Context Protocol): открытый стандарт Anthropic для подключения инструментов к любой модели. Сервер MCP = плагин: браузер, БД, Slack, GitHub — модель видит их как нативные инструменты.
